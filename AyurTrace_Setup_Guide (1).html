Create Database Tables (run SQL)
Creates users, batches, events tables
  1. In Supabase left sidebar → click SQL Editor
  2. Click "New query"
  3. Paste the SQL below → click "Run" (or Ctrl+Enter)
CREATE TABLE users (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  wallet_address TEXT UNIQUE NOT NULL,
  role TEXT NOT NULL CHECK (role IN ('collector','aggregator','processor','manufacturer','admin')),
  name TEXT, location TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE batches (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  batch_id TEXT UNIQUE NOT NULL,
  herb_name TEXT NOT NULL,
  herb_latin TEXT, quantity_kg DECIMAL,
  status TEXT DEFAULT 'collected',
  current_node TEXT DEFAULT 'collector',
  tx_hash TEXT, qr_code TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE events (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  batch_id TEXT NOT NULL REFERENCES batches(batch_id),
  node_type TEXT NOT NULL,
  actor_wallet TEXT NOT NULL, actor_name TEXT,
  latitude DECIMAL, longitude DECIMAL, location_name TEXT,
  notes TEXT, photo_url TEXT, tx_hash TEXT, block_number INTEGER,
  timestamp TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE batches ENABLE ROW LEVEL SECURITY;
ALTER TABLE events ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow all" ON users FOR ALL USING (true);
CREATE POLICY "Allow all" ON batches FOR ALL USING (true);
CREATE POLICY "Allow all" ON events FOR ALL USING (true);
✅ You should see: "Success. No rows returned" — tables created!

Deploy Frontend → Vercel (Free)
  1. Push project to GitHub: github.com → New repo → upload ayurtrace/ folder
  2. Go to vercel.com → Sign up with GitHub → "New Project"
  3. Import your GitHub repo → set Root Directory = frontend
  4. Add Environment Variables (from your frontend/.env)

🌍
Deploy Backend → Render.com (Free)
  1. Go to render.com → Sign up with GitHub → "New Web Service"
  2. Connect your GitHub repo → set Root Directory = backend
  3. Build command: npm install · Start command: npm start
  4. Add all backend .env variables in the Environment tab
  5. Click Deploy → get URL like ayurtrace-api.onrender.com
  6. Update frontend/.env: VITE_API_URL=https://ayurtrace-api.onrender.com
  7. Redeploy frontend on Vercel
⚠️ Render free tier sleeps after 15 min.
Fix: go to uptimerobot.com (free) → Monitor your Render URL every 5 min → backend stays awake!

Common Errors & Fixes
ErrorFix
MetaMask not foundInstall MetaMask extension and refresh page
CORS errorMake sure backend is running on port 5000
Supabase 401Check SUPABASE_SERVICE_KEY in backend/.env
Contract not foundRe-run deploy, update CONTRACT_ADDRESS in both .env files
GPS not workingUse localhost or HTTPS. Allow location permission in browser.
npm install failsDelete node_modules folder → run npm install again
Transaction failedGet more test MATIC from faucet. Check you're on Polygon Amoy.
Port already in useKill old processes: npx kill-port 5000 or npx kill-port 5173
